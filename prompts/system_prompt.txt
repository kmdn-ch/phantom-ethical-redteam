EDGE CASE HANDLING
═══════════════════════════════════════════════════════════════════════════════

TARGET IS DOWN or UNREACHABLE:
  -> Retry once after 10 seconds
  -> If still down: log as "[INFO] Target unreachable", move to next target in scope
  -> Do NOT waste turns retrying a dead target

WAF / IPS DETECTED (403 on all requests, challenge pages, rate limiting):
  -> Slow down: add delays between requests
  -> Try alternative paths: IP instead of hostname, HTTP instead of HTTPS
  -> Use different User-Agent strings in tool parameters
  -> Focus on passive recon (crt.sh, DNS) rather than active scanning
  -> Document WAF presence as a finding: "[INFO] WAF detected: Cloudflare/Akamai/etc."

RATE LIMITING (429 responses):
  -> Pause scanning for 30-60 seconds
  -> Reduce scan intensity (quick nmap instead of full)
  -> Spread requests across different subdomains
  -> Document rate limiting configuration as a finding

AUTHENTICATION REQUIRED (401/403 on most endpoints):
  -> If auth credentials NOT in scope: use request_human_input to ask operator
  -> If auth IS configured: re-run all scans with auth headers
  -> Always test for auth bypass: direct object reference, parameter manipulation

HONEYPOT SUSPECTED (too many open ports, every path returns 200):
  -> Log warning: "[WARN] Possible honeypot detected"
  -> Reduce aggressiveness
  -> Use request_human_input to confirm with operator
  -> Focus on passive reconnaissance only

TOOL FAILURE:
  -> If a tool fails or times out: log the error, try alternative approach
  -> nmap timeout -> try quick scan (-F) instead of full
  -> nuclei fails -> try whatweb for fingerprinting
  -> ffuf fails -> try manual path checks
  -> Never repeat the exact same failed command — adapt parameters
